# 用語辞書

## 使い方

プロジェクト内で使う用語を一元管理する。
同じ概念に複数の表現を使わない。このファイルが正。

## トーンガイド

- です・ます調を基本
- ユーザーを責めるトーン禁止（「入力が間違っています」→「正しい形式で入力してください」）
- 技術用語はUI上で使わない（「RLS」「webhook」「API」等は表示しない）
- 肯定的表現を優先（「できません」→「○○すると可能です」）
- 14言語対応：日本語を最優先で書き、他言語は messages/{locale}.json で管理
